# 图谱 2.0：知识星系 — 设计文档

**日期：** 2026-06-17
**状态：** 设计已确认，待写实现计划
**范围：** `/graph` 页面升级，分两期实现

## 目标

把 `/graph` 从"一颗会转的语义星球"升级成一张**会讲故事、能漫游、可解读的活星图**，
同时严守「简洁 + 实用 + 好玩，distill not decorate」原则：页面始终是**一张星图**，
不是一个仪表盘。

四个能力（一份设计、分期实现）：

| 代号 | 能力 | 角色 |
|---|---|---|
| **A** | 主题星系 | 静止态：节点按语义簇上色、漂浮中文星系名。是其余三者的底座。 |
| **B** | 时间回放 | 动词：按采集时间回放星图"从无到有"的生长。 |
| **D** | 语义漫游 | 动词：镜头自动沿语义边巡航，钩子做旁白，跨星系有播报。 |
| **E** | 语义路径 | 动词：选两颗星 → 点亮它们之间的概念桥 + LLM 解说。 |

（编号沿用头脑风暴时的六选项，C/F 本期不做。）

## UX 框架：静止态 + 临时动词

- **主题星系（A）永远是静止态** —— 一进 `/graph` 就看到上了色、带星系名的星图。
- **回放 / 漫游 / 路径（B/D/E）是临时触发的"动词"**，用完自动回到静止态。
- 入口是**底部一颗极简动词药丸**：`▶ 回放 · 🛸 逛逛 · 🔗 路径`。
- 同一时刻**只有一个动词激活**；激活时底部药丸变成该模式的控制器，`✕` 或完成即退出。
- 这是头脑风暴里对比三种框架后选定的"方案一"，气质最克制、最像"一张活的星图"。

## 架构原则（不变量）

- DB 是**派生视图**，文件仍是唯一真相源；所有新列/新表都可空、可由
  `python -m aishelf.db sync --rebuild` 重建。
- 新增逻辑沿用既有拆分风格：**纯逻辑模块 + 一个薄 DB 读写**（镜像 `graph.py` /
  `collide.py`）。
- 任何 LLM / embedding 失败都**降级、不崩溃**，保持读路径永不报错。
- 零新第三方依赖（numpy 已在）。
- embedding 未配置 ⇒ 无 embedding ⇒ 图谱本就为空 ⇒ 星系/动词自然为空，无需特判。

## 第 1 节：数据底座（A/B/D/E 共享）

四个能力都消费 `/api/graph` 的同一份富化数据。

### 当前载荷
- 节点：`{id, type, title, alias, degree, x, y, z}`
- 边：`{source, target, weight}`

### 富化后的载荷
- **节点新增**（全部源自现有 `items` 表，无新数据来源）：
  - `cluster: int | null` —— 所属星系 id
  - `collected_at: str` —— B 回放用（`items.collected_at` 已存在）
  - `hook: str` —— D 漫游旁白用（`items.hook` 已存在）
- **顶层新增** `clusters: [{id, name, color, size}]` —— 每个星系的中文名、颜色、成员数，
  供图例与漂浮标签使用。

### 新存储
- **新表 `clusters`**：`id INTEGER PRIMARY KEY, name TEXT, color TEXT`
- **`items` 加列** `cluster INTEGER`（可空）
- 二者均加入 `schema.py` 的 `SCHEMA_SQL`，与 `embedding/alias/hook` 同属"可空、可回填的派生列"。

### 聚类何时算、何时命名（关键取舍）
- 聚类是**全局**的（依赖整个语料），故在 `sync` 时计算。
- **仅当 embedding 集合变化时**（`embedded_ids` 非空 **或** 有 item 被 prune）才重算聚类
  + 重新命名 —— 复用与"边重算"完全相同的触发条件。
- **单纯改笔记不触发**重新聚类/命名：沿用现有"改笔记不重算 alias/hook"策略，
  也避免每次存笔记都花一次 LLM 调用。
- 颜色由**固定 8 色调色板**按 `cluster id` 决定（确定性，re-sync 不乱跳），由后端下发，
  保证图例与节点配色一致。

## 第 2 节：Phase 1 — A 主题星系

### 后端

**新模块 `aishelf/db/cluster.py`**（纯逻辑 + 一个薄写入）：

- `choose_k(n) -> int`：`clamp(round(sqrt(n/3)), 3, 8)`；`n < 3` 时返回 1（退化为单簇）。
  上限 8 与 `GRAPH_TOP_K` 审美一致，读起来就是"几大主题星系"。
- `kmeans(matrix, k, *, seed) -> labels`：numpy 实现，**固定 seed → 确定性**。
  在 **L2 单位化**后的 embedding 上聚类（等价于按余弦聚类，与图谱相似度口径一致）。
  小语料几十次迭代即收敛，无需 sklearn。
- `representatives(labels, matrix, ids, k, m=6) -> list[list[str]]`：每簇取离质心最近的
  m 个 item id（用于命名素材）。
- `recompute_clusters(con) -> dict[int, list[str]]`：读全部 embedding（复用
  `graph._load_groups` 的同维分组，避免换模型后的混维比较）→ `kmeans` →
  写回 `items.cluster`、重建 `clusters` 表（先 name 留空/占位，命名在 sync 层填）。
  返回 `{cluster_id: [代表标题…]}` 供命名。

> **备选方案（已否决）：** 基于边图的社区检测（modularity / 连通分量）。你的语料几乎全是
> AI，边图极易连成一个巨型连通块、分不开簇；k-means 在 embedding 空间能稳定切出均衡的簇。
> 故选 k-means。

**命名：`alias.py` 新增 `generate_cluster_names(clusters)`**
- `clusters: list[list[str]]`（每簇的代表标题），返回 `list[str] | None`。
- 复用现有批量 LLM 内核 `_generate_strings` 的同款结构：同一个 `ATLAS_CHAT_*` 门、
  同样的 JSON 数组解析、长度不匹配/异常 → `None` 降级。
- 提示：为每个簇起一个 **≤6 个汉字**的中文主题名，概括该簇内容。
- 失败时整体返回 `None`；调用方对缺名的簇回退为前端的"星系 N"。

**接入 `sync()`**（在现有边重算之后）：
```
if embedded_ids or stale:
    rep = cluster.recompute_clusters(con)          # 写 items.cluster + clusters 表骨架
    names = alias.generate_cluster_names([rep[i] for i in sorted(rep)])
    if names: 写回 clusters.name
    # 颜色：按 id 取调色板，可在 recompute_clusters 写入或读时计算
```
全部在同一事务内提交。任一步失败 → 记日志、保留已有行，不破坏 sync。

### 前端（`graph.html` 静止态升级）

- **按簇上色**：节点颜色取自 `clusters[].color`，取代当前 video/橙 + blog/蓝 的二元色。
- **类型仍可辨**：用 sprite 形状 / 描边区分 视频 vs 博客，使"主题（颜色）"与"类型（形状）"
  成为正交的两个维度。
- **漂浮星系标签**：每簇在其成员**质心**位置放一个比节点标签更大、半透明的 CSS2D 星系名标签。
- **图例进顶栏**：顶栏一行小色块 + 星系名，点击可只看某星系。现有的"视频/博客"类型筛选**保留**，
  作为正交的第二维过滤。
- **底部动词药丸（壳）**：放上 `▶ 回放 · 🛸 逛逛 · 🔗 路径`；Phase 1 中为占位（禁用态），
  Phase 2 接逻辑。
- Phase 1 交付一个**完整可用**的"上色 + 命名 + 图例 + 漂浮星系名"的静止星系图。

## 第 3 节：Phase 2 — B 回放 / D 漫游 / E 路径

三者均为**前端为主的动词**，消费 Phase 1 已富化的数据。共享约定：进入动词 → 接管底部条
+ 星图进入该模式；`✕` 或完成 → 回静止态；同一时刻仅一个动词激活；均无持久化。

### B 时间回放（纯前端）
- 底部条变**传送条**：`▶/⏸` + 可拖动进度条 + 当前日期标签。
- 进度条把 `collected_at` 的 min→max 映射成时间轴；playhead 扫过某节点日期时该节点
  从 opacity 0 渐显；边在两端节点都可见后才亮起。
- 一次 ▶ 约 **18s** 扫完；可拖 scrubber 手动定格任意时刻。

### D 语义漫游（纯前端）
- 🛸 启动：从一个节点出发，沿边做**偏向未访问、偏向高相似**的游走；镜头
  `controls.target` 缓动到每个节点，停留约 **3s**；底部字幕显示该节点 `alias` + `hook`。
- **跨星系时**字幕插一句 **「进入『<星系名>』」**（复用 A 的簇名，把漫游与星系串起来）。
- 点任意处 / `✕` 停止，回静止态。

### E 语义路径（前端找路 + 一个新流式后端）
- 🔗 → 提示"点两颗星"；依次点两个节点。
- **找路在前端**，跑在**当前可见的边**上（Dijkstra，cost = `1 - weight`）。
  高亮轨迹恰好是看得见的弧线，最直观。不连通 → 提示"这两颗星之间暂无语义通路"。
- 轨迹点亮、其余变暗。
- **LLM 解说（新后端）**：`POST /graph/path`，请求体为路径上**有序的 item id 列表**；
  服务端用现有 `db_search.get_by_ids` 水合标题/摘要，经新纯函数 `path.build_messages(items)`
  构造提示，`llm.stream_completion` 流式吐出"这条概念桥怎么从 A 一步步走到 B"——
  **完全镜像 `collide_chat`**（`StreamingResponse` + SSE）。
- **新模块 `aishelf/site/path.py`**：仅放纯 `build_messages(items_on_path)`，
  与 `collide.py` 的拆分风格一致。
- 这是 Phase 2 **唯一**的新后端路由；B/D 不碰后端。

## 第 4 节：测试

沿用 `tests/unit/` + fixtures，网络/LLM 全 mock（仅 `network` marker 下走真网）。

- **`cluster.py` 纯函数**：`kmeans` 确定性（固定 seed 下同输入同输出）、`choose_k` 边界
  （n<3→1、大 n 封顶 8）、`representatives` 选取正确。
- **`alias.generate_cluster_names`**：mock OpenAI，测 正常 / 数组长度不匹配→None / 异常→None
  降级（镜像现有 alias 测试）。
- **`sync` 集成**：embedding 变化才重算聚类、改笔记不触发、`clusters` 表被填充、
  命名失败不破坏 item 行。
- **`load_graph` 载荷**：节点带 `cluster/collected_at/hook`，顶层含 `clusters`，
  无 embedding 时整体为空。
- **`path.build_messages`**：纯函数断言 system/user 内容结构。
- **`POST /graph/path` 路由**：mock `llm.stream_completion`，断言 SSE 输出 +
  `get_by_ids` 水合；不连通在前端处理，不入后端测试。
- **前端三个动词的 three.js 交互无自动化测试** —— 与现有 `graph.html`（纯手验 JS）
  一致，是已知且接受的边界。
