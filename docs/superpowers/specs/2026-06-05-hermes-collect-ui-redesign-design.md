# Hermes 采集页 UI 重构 — 设计

日期：2026-06-05
范围：仅 `/collect` 页面（`templates/collect.html` 结构 + `static/style.css` 中 Hermes 相关样式）。
不动全站导航、其它页面、所有后端路由与前端 JS 行为。

## 问题

当前 `/collect` 页把口令栏、对话区、示例 chips、操作栏、定时采集表格/表单平铺成一长串，
权重相近、缺乏分组，显得拥挤、缺审美。

## 目标

简洁美观、信息分组清晰。视觉方向：**卡片分区**（用户选定）——把页面拆成几张独立白卡片，
靠间距与轻阴影分层；保留全站 iOS 蓝 `#0a84ff` 强调色以与其它页面一致。

## 布局（三张卡片，居中 max-width ~820px）

1. **Header 卡片**
   - 左：渐变蓝圆角方块头像「H」+ 标题「Hermes」+ 一行 lead 说明。
   - 下方一行：**采集口令** 紧凑输入框（带锁形提示文案），与标题同卡，弱化为次要信息。

2. **对话卡片**（页面焦点）
   - 顶部：可滚动对话区 `#log`；空态时显示居中提示文案（不再整块隐藏）。
   - 空态时展示「试试：」示例 chips（有对话后隐藏，沿用现 JS）。
   - 底部：composer（自增高 textarea + 圆形发送按钮）。
   - 卡片页脚细栏：左「清除对话」，右「查看视频 / 查看博客」链接。

3. **定时采集卡片**
   - 标题 + 一行 lead。
   - 任务列表：轻量表格（更大行距、更柔的分隔线）；空态友好提示。
   - 分隔线下：新增 / 编辑表单（名称 / 时间 / 需求 / 启用 / 保存）。

## 设计系统（本页）

- 卡片：白底，圆角 16px，`border 1px solid #ececf0`，阴影 `0 1px 3px rgba(0,0,0,.06)`，内距 20–24px。
- 卡片间距：20px。
- 字号层级：标题 22–24px、卡片小标题 15–16px、说明/次要 13px、正文 14–15px。
- 强调色 `#0a84ff`；聚焦态统一 `box-shadow 0 0 0 3px rgba(10,132,255,.12)`。

## 约束 / 不变量

- 所有元素 **id 与 class 钩子保持不变**（`log/examples/chatform/chatinput/chatsend/chatclear/
  collecttoken/schedform/sched-* /js-sched-*` 及 `tr` 的 data-* 属性），JS 与后端零改动。
- 仅重排 HTML 容器结构 + 重写 Hermes 相关 CSS。
- 模板 autoescape、`safe_url` 等安全约定不变。

## 验证

- `pytest`（现有 collect/schedules 视图测试须全绿）。
- 本地起站点，肉眼核对：空态、含对话、含定时任务三种状态布局正常，发送/示例/清除/
  定时增删改 交互不回归。
